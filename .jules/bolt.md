## 2024-05-18 - [Performance] Defer pretty-printing of raw JSON to UI on demand
**Learning:** In an immediate mode GUI framework like egui, putting expensive operations like `serde_json::from_str` or `to_string_pretty` inside the render loop causes catastrophic performance issues because it runs every frame. We fixed this by moving the serialization completely out of the UI state, waiting until the user explicitly requests the JSON view via a network request. This also saves memory.
**Action:** When shifting work from the background data processing layer (e.g., parsing feed posts) to the UI layer, always ensure the expensive work is done exactly once in the event/message handler and the result is cached in the application state. Never do the parsing/formatting directly inside the egui `show` closure.
